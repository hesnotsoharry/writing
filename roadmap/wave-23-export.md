# Wave 23 ‚ÄĒ Export sheet (Markdown / docx / PDF)

## Plan

### Status

DRAFT ¬∑ target v0.x (minor ‚ÄĒ feature wave) ¬∑ drafted 2026-06-04.

### Goal

After this wave the app has a working Export feature under `src/features/export/`: a `<ExportOverlay>` sheet that takes a scope (single scene, a chapter, or the whole manuscript) and writes the scene prose out as Markdown, `.docx`, or PDF. Prose is pulled through the existing Yjs path (`sceneDocStore.load ‚Üí applyEncoded ‚Üí extractPlainText`), assembled in binder order, formatted by pure renderer-side functions, and handed to the OS. The format/collect logic is fully unit-tested without a Tauri runtime; the overlay calls an injected `onSave` prop and falls back to an in-lane Blob-download so the feature is demonstrable before the lead wires the portable save path. No `src-tauri/` and no `src/App.*` are touched by this lane.

### Scope

**In scope:**

- `src/features/export/exportCollect.ts` ‚ÄĒ pure async `collectBlocks(scope, targetId, tree, store) ‚Üí Promise<{ blocks: string[]; suggestedTitle: string }>`. Resolves scope to an ordered scene list off the `BinderTree`, loads each via `sceneDocStore`, decodes with `applyEncoded` + `extractPlainText`, prepends chapter-heading lines for manuscript scope, and returns a filesystem-sanitized `suggestedTitle`.
- `src/features/export/exportFormatters.ts` ‚ÄĒ pure `toMarkdown(blocks, title) ‚Üí string`, `toDocx(blocks, title) ‚Üí Promise<Uint8Array>`, `toPdf(blocks, title) ‚Üí Promise<Uint8Array>`. PDF paginates via `jspdf` `splitTextToSize` + manual page breaks, yielding to the event loop between pages.
- `src/features/export/blobDownloadSave.ts` ‚ÄĒ in-lane `SaveCallback` fallback using `URL.createObjectURL` + a programmatic anchor click (works in the Tauri v2 webview per wry ‚Č•0.53).
- `src/features/export/Export.tsx` ‚ÄĒ the `<ExportOverlay>` component: `.scrim`/`.sheet` canon overlay, format picker (Markdown / docx / PDF), scope label, an Export action that runs collect ‚Üí format ‚Üí save with a transient "Generating‚Ä¶" state and error surface.
- `src/features/export/index.ts` ‚ÄĒ barrel export of `ExportOverlay` + the `ExportOverlayProps`/`ExportScope`/`SaveCallback` types.
- Additive deps in `package.json`: `jspdf` `^4.2.1`, `docx` `^9.7.1` (lockfile regenerated by the lead on merge).
- Vitest suites for `exportCollect` and `exportFormatters` (via `InMemorySceneDocStore`), plus a behavior test for `Export.tsx` (format pick ‚Üí save called with right args; `onSave` omitted ‚Üí fallback used).

**Out of scope:**

- Any `src-tauri/` change (Rust crates, `lib.rs`, capabilities) and the portable `@tauri-apps/plugin-dialog` + `plugin-fs` `onSave` implementation ‚ÄĒ **lead owns these on integration** (see Locked Decision C). Deferral path: lead's integration commit on `master`.
- Wiring the three `onExport` toast stubs (`BinderCrud.tsx`, `Corkboard.tsx`) and the TitleBar Export button, and widening `OverlayStackProps`/`App.overlays.tsx` to pass context props ‚ÄĒ **lead owns** (`src/App.*` is frozen for this lane). Deferral path: lead's integration commit.
- Rich-formatting export (bold/italic/headings inside prose). `extractPlainText` is plain-text-only today; styled export waits until the editor's rich structure is exposed through the export pipeline. Deferral path: a later wave once formatting survives extraction.
- Export presets / page-setup options (margins, font, title page). Out for v1. Deferral path: follow-up if requested.

### Acceptance criteria

- [ ] `collectBlocks("scene", sceneId, tree, store)` returns `{ blocks: [<that scene's extractPlainText>], suggestedTitle: <sanitized scene title> }`.
- [ ] `collectBlocks("chapter", folderId, tree, store)` returns the chapter's scenes in `sort_order`, blocks in that order, `suggestedTitle` from the chapter folder title.
- [ ] `collectBlocks("manuscript", projectId, tree, store)` returns every chapter's scenes in binder order, each chapter preceded by a heading-line block; short pieces included.
- [ ] `suggestedTitle` contains none of `\ / : * ? " < > |` and is non-empty (falls back to a default when the source title is blank).
- [ ] `toMarkdown(blocks, title)` returns a string beginning with `# <title>` and blocks separated by a blank line.
- [ ] `toDocx(blocks, title)` resolves to a non-empty `Uint8Array` whose first bytes are the ZIP signature `PK\x03\x04` (docx is a zip).
- [ ] `toPdf(blocks, title)` resolves to a `Uint8Array` whose first bytes decode to `%PDF`.
- [ ] `toPdf` paginates: a single block of ‚Č•5000 words yields a document with `getNumberOfPages() > 1` (no silent truncation).
- [ ] `blobDownloadSave(filename, data, mime)` creates an object URL and triggers an anchor click with `download === filename` (verified via spy).
- [ ] `<ExportOverlay>` renders `.scrim` + `.sheet` with a Markdown/docx/PDF picker and Export + Close controls; clicking Export calls the resolved save callback with `(suggestedFilename, data, mime)` matching the picked format.
- [ ] When `onSave` is omitted, `<ExportOverlay>` routes the save through `blobDownloadSave` (verified via spy).
- [ ] `index.ts` exports `ExportOverlay`, `ExportOverlayProps`, `ExportScope`, `SaveCallback`.
- [ ] Gates green in the worktree: `npm run lint`, `npx tsc --noEmit`, `npm run test` on the touched export tests.

## Locked decisions

> Decided via the decision-review cell (`sonnet-architect` ‚Üí `sonnet-adversarial-reviewer` `Posture: attack-decision` ‚Üí orchestrator adjudication). Research sidecar: `roadmap/wave-23-export-research.md`.

### Decision A: PDF engine

**Context:** Export must emit PDF, but this lane cannot edit `src-tauri/` ‚ÄĒ a Rust/sidecar PDF path is out of reach, and the engine must run + be tested without a Tauri runtime.
**Pick:** `jspdf` `^4.2.1`, renderer-side. Plain-text pagination via `doc.splitTextToSize` + manual Y-tracking/`addPage`, `await`-ing a macrotask between pages so a long manuscript doesn't freeze the main thread.
**Rationale:** Only in-lane option; v4.2.1 ships a Node build so formatters are vitest-testable; FAIR prose quality is acceptable for a plain-text export. Reversible ‚ÄĒ `toPdf` is the only jspdf consumer, so a future Rust-sidecar swap is contained behind that function.
**Consequences:** ~95 KB renderer bundle; a `paginatePlainText` helper; "Generating‚Ä¶" state for large exports; PDF output is unstyled plain text (flagged to Cole ‚ÄĒ see Result/handoff).
**Enforcement:** advisory-only (gates: eslint/tsc/vitest on `exportFormatters.ts`; the multi-page acceptance criterion is the behavioral guard).

### Decision B: docx engine

**Context:** Export must emit a `.docx` that opens in Word/LibreOffice, generated in-lane and node-testable.
**Pick:** `docx` `^9.7.1`. `toDocx` returns `Uint8Array` via `new Uint8Array(await Packer.toBlob(doc).arrayBuffer())` in the browser path; tests use `Packer.toBuffer` and wrap to `Uint8Array`. One paragraph per block.
**Rationale:** Mature, pure-JS, browser+node, emits a real OOXML zip; uniform `Uint8Array` transport across the `onSave` boundary avoids a `Blob`-vs-`Uint8Array` fork (the adversarial review's hidden-coupling flag ‚ÄĒ resolved by locking `Uint8Array`).
**Consequences:** one dep added; output is plain paragraphs (no styling, matching the plain-text source).
**Enforcement:** advisory-only (the ZIP-signature + non-empty acceptance criteria are the guard).

### Decision C: file-save mechanism + component prop contract

**Context:** Saving a file needs `tauri-plugin-dialog` + `tauri-plugin-fs` (the only cross-platform-correct path incl. mobile), but those live in `src-tauri/` which this lane cannot touch; the feature must still be demonstrable in-lane.
**Pick:** `<ExportOverlay>` generates content and calls an **optional** `onSave?(suggestedFilename, data, mime): Promise<void>` prop. When absent it falls back to the in-lane `blobDownloadSave` (Blob-URL anchor download ‚ÄĒ confirmed working in the Tauri v2 webview, wry ‚Č•0.53). The **lead** implements the portable `onSave` with `@tauri-apps/plugin-dialog`.`save()` + `@tauri-apps/plugin-fs`.`writeFile`/`writeTextFile` on integration. `window.showSaveFilePicker` was rejected ‚ÄĒ no macOS WKWebView support, needs `useHttpsScheme` on Windows; non-portable for a Windows-now/mobile-later app.
**Rationale:** Keeps the lane strictly inside `src/features/export/*`, makes the feature demonstrable immediately via the fallback, and reserves the cross-platform-correct native Save-As for the integration owner ‚ÄĒ one seam (`onSave`) serves both paths. `Uint8Array | string` transport keeps the boundary format-agnostic (lead branches on `typeof data` for text vs binary write; filename carries the extension so the dialog pre-fills correctly).
**Consequences (lead's integration work ‚ÄĒ flag in handoff):** add `tauri-plugin-dialog` + `tauri-plugin-fs` to `Cargo.toml` + `lib.rs`; grant `dialog:allow-save` + `fs:allow-write-file`/`fs:allow-write-text-file` in capabilities; add `@tauri-apps/plugin-dialog` + `@tauri-apps/plugin-fs` npm deps; implement the real `SaveCallback` and pass it as `onSave` at the `App.overlays` integration site; pass `defaultPath = suggestedFilename`.
**Prop contract:**
```ts
export type ExportScope = "scene" | "chapter" | "manuscript";
export type ExportData = string | Uint8Array;
export type SaveCallback = (suggestedFilename: string, data: ExportData, mime: string) => Promise<void>;
export interface ExportOverlayProps {
  projectId: string;
  scope: ExportScope;
  targetId: string;            // scene.id | folder.id | projectId (manuscript)
  sceneDocStore: SceneDocStore;
  tree: BinderTree;
  onClose: () => void;
  onSave?: SaveCallback;       // lead supplies on integration; absent ‚áí blobDownloadSave fallback
}
```
**durable: candidate** ‚ÄĒ the `onSave`-prop seam and renderer-side-export pick are likely cited by any future export/import or mobile-save wave.
**Enforcement:** advisory-only (the `onSave`-omitted-falls-back-to-blobDownloadSave acceptance criterion is the behavioral guard; lane-ownership is enforced by the merge review + the coordination doc's disjoint-dir model).
