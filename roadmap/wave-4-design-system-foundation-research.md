## 2. Import Patterns in Vite/React

### Static Font Imports

Install the static package:
```bash
npm install @fontsource/literata
```

Import a specific weight and style:
```javascript
// In your main.tsx or a styles file (e.g., tokens.css.ts)
import "@fontsource/literata/400.css";      // Regular weight
import "@fontsource/literata/700.css";      // Bold weight
import "@fontsource/newsreader/300.css";    // Light weight
import "@fontsource/newsreader/400-italic.css"; // Italic style
```

The CSS import registers the `@font-face` declaration automatically. The font becomes available under its canonical `font-family` name (e.g., `'Literata'`, `'Newsreader'`) — no additional CSS or JS wiring needed beyond the import.

**Where to import:** Conventionally in your app's entry file (`src/main.tsx`) or a dedicated styles file that is imported early in the boot sequence. The import order doesn't matter (fonts are registered globally).

### Variable Font Imports

Install the variable package:
```bash
npm install @fontsource-variable/literata
```

Import the variable font:
```javascript
// Single import covers all weights (200–1000 by default)
import "@fontsource-variable/literata";

// OR with specific axis variants
import "@fontsource-variable/literata/wght-italic.css";  // Weight + italic axis
import "@fontsource-variable/literata/full.css";         // All available axes
```

The import registers the variable `@font-face`. The font is available under its canonical name with the `Variable` suffix (e.g., `'Literata Variable'`, `'Newsreader Variable'`).

**Critical gotcha (see §4 below):** Variable font family names include the `Variable` suffix in the CSS `font-family` declaration. If tokens.css hardcodes `font-family: 'Literata'`, it won't match the variable package's registered `'Literata Variable'` — you must update the token.

## 3. Variable vs. Static Recommendation for Offline Desktop App

**Recommendation: Variable fonts for Phase 1 (Tauri bundling).**

**Tradeoff analysis (from Fontsource FAQ):**

| Axis | Static (per-weight) | Variable (single file) |
|---|---|---|
| **Bundle size** | 4 weights × ~25 KB = ~100 KB | 1 file with all weights = ~35–50 KB |
| **Weight coverage** | Only imported weights available | 200–1000 weight range in one file |
| **Flexibility** | Fixed weights; no tweaking | `font-variation-settings` enables weight sliding |
| **File count** | Multiple CSS + font files | Single CSS + font file |
| **Build complexity** | Import each weight separately | Single import covers the spectrum |

**For a desktop Tauri app bundling fonts locally:**
- Variable fonts are the clear win: one file per typeface, all weights, smaller total footprint, and future-proof (can adjust weight on-the-fly via CSS if design evolves).
- If a font family has only 1–2 weights in use (e.g., IBM Plex Mono 400 + 700), static is acceptable; but Literata and Newsreader are typography staples and benefit from the weight range.

**Caveat:** IBM Plex Mono variable availability is unverified in this research pass — check npm before assuming it exists.

## 4. Font-Family Naming Gotcha

**Static and variable packages register different `font-family` names in @font-face.**

### Static Package
```css
/* @font-face generated by @fontsource/literata/400.css */
@font-face {
  font-family: "Literata";
  src: url(...) format("woff2");
  font-weight: 400;
}
```

CSS uses: `font-family: 'Literata'`

### Variable Package
```css
/* @font-face generated by @fontsource-variable/literata */
@font-face {
  font-family: "Literata Variable";
  src: url(...) format("woff2-variations");
}
```

CSS uses: `font-family: 'Literata Variable'`

**Impact on tokens.css:**

If tokens.css currently hardcodes:
```typescript
export const typography = {
  serif: { fontFamily: "'Literata', serif" },
  sansSerif: { fontFamily: "'Newsreader', sans-serif" },
};
```

Switching to variable packages requires updating the token names:
```typescript
export const typography = {
  serif: { fontFamily: "'Literata Variable', serif" },      // CHANGED
  sansSerif: { fontFamily: "'Newsreader Variable', sans-serif" }, // CHANGED
};
```

**Mitigation:** Use CSS @supports rule to fallback gracefully (for platforms without variable font support):
```css
body {
  font-family: "Literata", serif;  /* Fallback */
}

@supports (font-variation-settings: normal) {
  body {
    font-family: "Literata Variable", serif;
    font-variation-settings: "wght" 400;
  }
}
```
